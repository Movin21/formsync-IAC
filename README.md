# formsync IAC

Industry-standard Terraform + Ansible to provision a `t2.large` EC2 instance on AWS and configure Docker Engine + Docker Compose plugin.

## Layout

```
.
├── terraform/                # AWS infrastructure (VPC, SG, key pair, EC2)
│   ├── versions.tf
│   ├── providers.tf
│   ├── variables.tf
│   ├── data.tf
│   ├── network.tf
│   ├── security.tf
│   ├── key_pair.tf
│   ├── ec2.tf
│   ├── outputs.tf
│   └── terraform.tfvars.example
└── ansible/                  # Configuration management
    ├── ansible.cfg
    ├── requirements.yml
    ├── group_vars/all.yml
    ├── inventory/hosts.ini   # generated by terraform apply
    ├── playbooks/site.yml
    └── roles/docker/         # idempotent Docker + Compose role
```

## Prerequisites

- Terraform `>= 1.5`
- Ansible `>= 2.14` (run from Linux/macOS/WSL — Ansible does not run natively on Windows)
- AWS credentials configured (`aws configure` or environment variables)
- An SSH client

## Usage

### 1. Provision infrastructure

```bash
cd terraform
cp terraform.tfvars.example terraform.tfvars   # edit as needed
terraform init
terraform plan -out tfplan
terraform apply tfplan
```

This creates: VPC, public subnet, internet gateway, route table, security group, RSA key pair (private key written locally with mode `0400`), and a `t2.large` Ubuntu 22.04 EC2 instance. It also writes `../ansible/inventory/hosts.ini`.

### 2. Configure Docker via Ansible

```bash
cd ../ansible
ansible-galaxy collection install -r requirements.yml
ansible -i inventory/hosts.ini docker_hosts -m ping
ansible-playbook playbooks/site.yml
```

The `docker` role:
- adds Docker's official apt repo
- installs `docker-ce`, CLI, containerd, buildx, and the **Compose plugin** (`docker compose`, not the legacy `docker-compose`)
- writes `/etc/docker/daemon.json` with sane log limits and `live-restore`
- adds `ubuntu` to the `docker` group
- deploys a sample `docker-compose.yml` to `/opt/formsync` and brings the stack up

### 3. Verify

```bash
ssh -i terraform/formsync-docker-key.pem ubuntu@$(terraform -chdir=terraform output -raw instance_public_ip)
docker compose -f /opt/formsync/docker-compose.yml ps
curl http://$(terraform -chdir=terraform output -raw instance_public_ip)/
```

### 4. Tear down

```bash
cd terraform
terraform destroy
```

## Hardening checklist before production

- [ ] Restrict `ssh_allowed_cidrs` to your office/VPN range — do not leave `0.0.0.0/0`
- [ ] Enable the S3 + DynamoDB remote backend (commented in `versions.tf`)
- [ ] Replace the local key pair with one stored in AWS Secrets Manager / SSM
- [ ] Attach an IAM instance profile if the workload needs AWS API access
- [ ] Move SSH behind SSM Session Manager and remove port 22 entirely
- [ ] Enable CloudWatch agent + alarms on CPU, disk, and status checks
- [ ] Switch `t2.large` to `t3.large` or `m6i.large` for better price/performance
